# إعدادات Supabase - تفعيل التسجيل الفوري (بدون تأكيد بريد)

## المشكلة الحالية
- عند التسجيل، Supabase يطلب تأكيد البريد الإلكتروني
- هذا يمنع المستخدم من تسجيل الدخول مباشرة

## الحل - تعطيل التحقق من البريد (للاختبار/التطوير)

### خطوات تفعيل التسجيل الفوري:

#### 1. في لوحة التحكم Supabase:
```
Dashboard → Authentication → Providers → Email
```

#### 2. ابحث عن الإعدادات التالية:
- **"Confirm email"** → غيّرها من `enabled` إلى `disabled`
- **"Double confirm changes"** (إن ظهرت) → اضغط confirm

#### 3. اضغط **Save**

### النتيجة:
✅ عند التسجيل، المستخدم سيتم تسجيل دخوله مباشرة
✅ سيظهر اسمه وصورته بدل أزرار Login
✅ سيتم توجيهه تلقائيًا إلى صفحة `/user`

## ملاحظات أمنية:
⚠️ **هذا الإعداد فقط للتطوير والاختبار**
- في الإنتاج (Production)، يجب تفعيل التحقق من البريد

## طريقة بديلة (إذا لم تستطع تعديل الإعدادات):

استخدم **Supabase SQL** لتأكيد البريد يدويًا:

```sql
-- في Supabase SQL Editor
UPDATE auth.users
SET email_confirmed_at = now(),
    confirmed_at = now()
WHERE email = 'user@example.com';
```

ثم حاول تسجيل الدخول.
